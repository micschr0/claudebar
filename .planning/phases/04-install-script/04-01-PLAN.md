## Phase Goal

**As a** new user on any supported OS/arch, **I want to** run one command to install claudebar, **so that** I get a working statusline regardless of whether Rust is installed on my machine.

<objective>
Rewrite `install.sh` as a three-tier fallback installer: (1) download prebuilt binary from GitHub Releases + verify SHA256; (2) fall back to `cargo install --path .` if no prebuilt matches; (3) fall back to `statusline-command.sh` if cargo is unavailable.

Purpose: New users who run `curl -fsSL .../install.sh | bash` currently get no binary if they lack Rust â€” the rewrite makes installation work on any supported platform without the Rust toolchain.

Output: Rewritten `install.sh` that passes `bash -n` and `shellcheck` at warning severity, implements INST-01 through INST-05, and preserves all existing `settings.json` patching and done-block logic.
</objective>

Function `detect_target()` (implements INST-01): declares `local os arch` separately (SC2155), assigns with `os=$(uname -s)` and `arch=$(uname -m)`. Case on arch: `arm64|aarch64` â†’ `arch="aarch64"`, `x86_64` â†’ `arch="x86_64"`, `*` â†’ `printf ''` and return. Function `sha256_of()` (used by INST-02): single arg `"$1"`. `if command -v sha256sum >/dev/null 2>&1; then sha256sum "$1" | awk '{print $1}'; else shasum -a 256 "$1" | awk '{print $1}'; fi`. No local variables needed.

Function `verify_checksum()` (implements INST-02 security): args `file="$1"`, `archive_name="$2"`, `sums_file="$3"`. Declare `local expected actual` on one line. If actual != expected: call `red "SHA256 mismatch â€” aborting (download may be tampered or corrupted)"`, then `red "  Expected: ${expected}"`, then `red "  Actual:   ${actual}"`, then `return 1`. Otherwise call `green "SHA256 verified"` and `return 0`.

Function `download_prebuilt()` (implements INST-02): arg `target="$1"`. Declare `local tag archive url sums_url` on one line. Assign `tag=$(fetch_latest_tag)`. If tag is empty: call `bold "No prebuilt release found â€” falling back to cargo build"` and `return 1`. Set `archive="claudebar-${tag}-${target}.tar.gz"`, `url="https://github.com/micschr0/claudebar/releases/download/${tag}/${archive}"`, `sums_url="https://github.com/micschr0/claudebar/releases/download/${tag}/SHA256SUMS.txt"`. Call `bold "Downloading ${archive}..."`. Call `verify_checksum "${TMPDIR_WORK}/${archive}" "$archive" "${TMPDIR_WORK}/SHA256SUMS.txt"` â€” if it fails, `return 1` (checksum failure is fatal, not a fallback trigger). Extract: `tar -xf "${TMPDIR_WORK}/${archive}" -C "${TMPDIR_WORK}/"`. Move only the binary: `mv "${TMPDIR_WORK}/claudebar" "$BIN_DEST"`. Set permissions: `chmod +x "$BIN_DEST"`. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user shell â†’ GitHub Releases HTTPS | Untrusted network; TLS is the transport guard |
| GitHub Releases HTTPS â†’ install.sh | Downloaded archive and SHA256SUMS.txt are untrusted until checksum verified |
| SHA256SUMS.txt content â†’ local verification | Checksum file itself downloaded over HTTPS; not signed â€” trust anchored in TLS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | curl download of archive + SHA256SUMS.txt | mitigate | HTTPS-only (`curl -fsSL`), TLS verified by default; never strip with `--insecure` |
| T-04-02 | Tampering | Downloaded binary before extraction | mitigate | `verify_checksum()` compares SHA256 of downloaded archive against SHA256SUMS.txt before any extraction or installation; mismatch is fatal (`return 1`, exits script) |
| T-04-03 | Tampering | SHA256SUMS.txt itself | accept | File is delivered over HTTPS (same TLS anchor as the binary); signing SHA256SUMS.txt with a GPG key is a v2 hardening â€” deferred |
| T-04-04 | Elevation | tar extraction of archive containing non-binary files | mitigate | `tar -xf` to isolated `$TMPDIR_WORK`; only `mv "${TMPDIR_WORK}/claudebar"` is moved to BIN_DEST â€” README.md and LICENSE remain in temp dir and are deleted by EXIT trap |
| T-04-05 | Tampering | Path traversal in tar archive | mitigate | Extraction to isolated `$TMPDIR_WORK` (not `/` or `$HOME`); only the known relative path `claudebar` is moved out |
| T-04-06 | Tampering | Silent fallback bypassing checksum | mitigate | `verify_checksum` failure calls `return 1` from `download_prebuilt`, which propagates up â€” download_prebuilt failure does NOT set COMMAND_VALUE, so Tier 2 cargo runs instead; checksum bypass is not possible without script modification |
| T-04-SC | Tampering | No new npm/pip/cargo installs in this phase | accept | Phase 4 modifies only a bash script; no package manager installs; no Package Legitimacy Audit needed |
</threat_model>
